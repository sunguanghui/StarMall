# 🐛 缺陷修复任务清单 (Bug Fixes List)

## 第一部分：UI 与表单交互修复 (UI & Forms)
**缺陷 1：左侧菜单排序不当**
- **问题描述**：「航行指南」菜单目前的位置过于靠上。
- **修复要求**：请修改 `MainLayout.vue` 或相关的菜单渲染组件，将「航行指南」移动到「管理中心」的**正下方**。

**缺陷 2：用户编辑表单的必填校验错误**
- **问题描述**：在「管理中心-用户管理」编辑已有用户时，如果不修改密码（留空），点击保存会触发“请输入密码”的强制校验拦截，导致无法保存其他信息的修改。
- **修复要求**：请修改用户管理的编辑表单逻辑（如 `Users.vue`）。当动作为“编辑 (Edit)”时，密码字段应为**非必填项**（仅当用户输入了新密码时才提交修改，留空则代表不修改密码）。

## 第二部分：数据加载异常修复 (Data Fetching Errors)
**缺陷 3 & 4：任务大厅页面崩溃 (TypeError: Cannot read properties of undefined)**
- **问题描述**：打开「任务大厅」的“每日日常”或“我的记录”页面时，页面白屏一直转圈，控制台报错：`TypeError: Cannot read properties of undefined (reading 'filter')`。这通常是因为前端尝试对一个未定义 (undefined) 或 null 的数组执行 `.filter()` 操作。
- **修复要求**：请检查 `Tasks.vue` 或相关子组件的数据加载逻辑。在接收后端 API 返回的任务列表或记录列表数据时，必须增加**兜底保护机制**（例如：`const list = response.data || []`），确保在尝试使用 `.filter()`、`.map()` 之前，该变量绝对是一个合法的数组。

**缺陷 5：后台任务管理界面加载失败**
- **问题描述**：打开「管理中心-任务管理」界面时，页面直接提示“加载记录失败”。
- **修复要求**：请检查 `AdminTasks.vue` 的 API 请求逻辑以及后端的 `GET /api/tasks` 相关路由。可能是因为缺少某些字段处理或权限校验失败导致的 500/403 错误。请排查并修复该数据加载链路。

## 第三部分：移动端适配与 PWA 修复 (Mobile & PWA)
**缺陷 6：PWA 独立窗口失效与安装提示缺失**
- **问题描述**：在手机端添加到主屏幕后打开，依然有浏览器地址栏；且网页访问时没有弹出 PWA 安装提示。
- **修复要求**：请重新检查 `vite.config.js` 中的 `vite-plugin-pwa` 配置。确保 `manifest` 中包含了 `display: "standalone"` 或 `display: "fullscreen"`；并确保入口文件（如 `main.js` 或 `App.vue`）中正确调用了 PWA 的注册和安装提示逻辑。

**缺陷 7：移动端表格文字严重挤压**
- **问题描述**：在手机端，首页的「最近发放的星辰币」和「最近兑换记录」表格列数过多，导致文字相互挤压、难以阅读。
- **修复要求**：对于移动端屏幕（通过 CSS 媒体查询或 Element Plus 的响应式属性），请将这两个原生 `el-table` 隐藏，并降级渲染为基于 `el-card` 或列表项 (List Item) 的**卡片堆叠布局**。

**缺陷 8：移动端管理后台表格无法左右滑动**
- **问题描述**：在手机端进入用户管理、商品管理等后台页面，由于表格列很多，右侧的“操作”列被固定或遮挡，且整个表格容器无法左右滑动查看。
- **修复要求**：请为这些后台管理页面的 `el-table` 外层容器添加包裹层，并设置 `overflow-x: auto;` 和 `-webkit-overflow-scrolling: touch;`，允许用户在小屏幕上横向滚动表格内容。
