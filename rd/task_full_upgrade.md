# 需求明细 (Requirements to Analyze)

## 阶段一：UI 与文案深度主题化（星际探索风）
系统目前使用的是“大拇哥”和“积分”主题。我们需要将其彻底改造为适合孩子的“星际探索”主题。
- **UI 文本全局替换 (仅限视图层)**：
  - "大拇哥积分商城" 或 "大拇哥商城" -> 替换为 "星途补给站"
  - "大拇哥" -> 替换为 "星辰币"
  - "积分" -> 替换为 "能量"
- **Element Plus 图标 (Icon) 替换**：
  - 将代表“单倍奖励”（原单大拇哥/1分）的图标替换为 `<Star />` (星星) 图标。
  - 将代表“双倍奖励”（原双大拇哥/5分）的图标替换为 `<Ship />` (火箭/飞船) 图标。

## 阶段二：趣味化与核心机制升级 (P0)
### 1. 积分扣除/红牌警告机制 (Point Deduction)
- **后端逻辑**：修改发放奖励的 API 路由和数据库模型，使其支持接收并正确计算**负数**能量值（扣分功能）。
- **前端管理员端**：在发分表单（如 `Thumbs.vue`）中，增加一个切换选项（单选框或 Switch），允许在“增加能量 (加分)”和“扣除能量 (扣分)”之间切换。如果选择扣除，则确保向后端发送负整数。
- **前端用户端**：在积分流水历史记录中（如 `MyPoints.vue`, `Dashboard.vue`），使用 Vue 的动态绑定（`:class` 或 `:style`），将值为负数的记录以**红色字体**显著渲染。

### 2. 盲盒兑换功能 (Blind Box)
- **业务逻辑**：在 `products` (商品表) 中新增一个 `is_blind_box` 布尔字段。前端针对盲盒商品展示特殊的卡片样式。
- **兑换逻辑**：当用户兑换盲盒商品时，后端从一个预设的奖池（可直接在配置文件或常量中预设几个基础小奖品）中随机抽取（`random.choice`）一个具体奖励，并将抽取结果记录到兑换历史 `exchange_records` 中。

### 3. 星际心愿单 (Wishlist)
- **业务逻辑**：新增 `wishlists` 数据库表（字段包含 `user_id`, `title`, `expected_points`, `status`）。
- **前端逻辑**：普通用户可提交心愿单；管理员可在后台审核，点击“批准”后，系统自动将该心愿写入 `products` 表，转化为正式上架的商品。

## 阶段三：数据可视化与家庭互动 (P1)
### 1. 舰长寄语 (Parent Message)
- **数据库**：在 `thumbs_records` 表增加 `parent_message` (TEXT) 字段。
- **前后端逻辑**：管理员发分时可填写留言文本。用户在前端查看流水明细时，如果有寄语，需使用气泡框或温馨提示条展示。

### 2. 能量收集趋势图表 (Visual Dashboard)
- **前端逻辑**：在用户的仪表盘 (`Dashboard.vue`) 引入 ECharts 库，基于用户的积分流水历史数据，渲染一个展示能量累计趋势的折线图。
