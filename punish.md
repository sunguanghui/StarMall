# 角色与上下文 (Role & Context)
你现在是 Claude Code。我们的「星途补给站」目前已经有了基础的扣除能量（扣分）功能，但目前的扣分力度太弱（只能固定扣 1 分），起不到约束和惩罚的作用。
现在的核心任务是：**全面升级扣分机制，引入“阶梯式大额惩罚”与“自定义惩罚”功能，并在视觉上强化警示感。**

# ⚠️ 阶梯式惩罚机制升级指南

## 1. 后端逻辑校验 (Backend Validation)
- 检查处理发放/扣除能量的 API 路由（如 `app.py` 中的 `thumbs` 或相关接口）。
- 确保系统不仅能接收 `-1`，还能完美接收并处理任何负整数（如 `-5`, `-10`, `-50`），并且可用能量 (`available_points`) 允许被扣成负数或最低归零（请结合当前业务逻辑处理边界情况，如果能量不足，允许直接扣到 0）。

## 2. 前端管理员端改造 (`frontend/src/views/admin/Thumbs.vue` 或相关发分组件)
- **重构扣分面板**：当管理员切换到“扣除能量”模式时，抛弃单一的按钮，改为提供一组具有强烈视觉冲击力（如 `el-button type="danger"`）的快捷选项：
  - **陨石撞击 (-2 能量)**：适用于轻微犯错。
  - **星暴气流 (-5 能量)**：适用于中度违规。
  - **黑洞吞噬 (-10 能量)**：适用于严重违纪。
- **自定义黑洞**：增加一个自定义数字输入框 (`el-input-number`，设置 `max="-1"`，步长为 1)，允许家长手动输入任意负数进行大额惩罚。

## 3. 前端用户端视觉强化 (Visual Deterrence)
- 在用户的能量流水记录页 (`MyPoints.vue` 等) 中，针对负数记录，除了使用红色字体外，请给这种大额惩罚记录（如扣除 5 分及以上）加上醒目的图标（如 `<Warning />` 或一个燃烧的流星图标），让孩子一眼就能看到“大出血”的惨痛教训。
