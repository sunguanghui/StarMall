# 🔐 多级权限与家庭协同重构指南

## 1. 数据库全面升级 (Database Schema)
请使用 `ALTER TABLE` 或更新 ORM 模型完成以下表结构改造：
- **`users` 表**：新增布尔字段 `is_super_admin` (默认 false)，用于区分超级管理员（舰长：爸爸/妈妈）和普通管理员（领航员：长辈/老师）。
- **`tasks` 表 (任务模板)**：
  - 新增 `reviewer_id` (整数，外键关联 `users.id`)：代表该任务的专属审批人。
  - 新增 `created_by` (整数，外键关联 `users.id`)：记录任务的创建者。
- **`thumbs_records` 表 (积分流水)**：
  - 新增 `admin_id` (整数，外键关联 `users.id`)：明确记录这笔能量是由哪位管理员操作发放的（历史存量数据可默认设为超级管理员 ID）。

## 2. 后端 API 逻辑与权限隔离 (Backend API)
- **创建任务 (POST /api/tasks)**：
  - **超级管理员**：允许传入 `reviewer_id`（指定具体的某个 admin 作为审批人）。
  - **普通管理员**：后端**强制**将 `created_by` 和 `reviewer_id` 都设置为当前操作者的 ID，忽略前端传入的其他审批人参数。
- **编辑/删除任务 (PUT/DELETE /api/tasks/<id>)**：
  - 普通管理员**只允许**修改或删除 `created_by` 等于自己 ID 的任务。越权操作返回 403。
- **获取任务列表 (GET /api/tasks)**：
  - 超级管理员：返回所有任务。普通管理员：仅返回 `created_by` 是自己的任务。
- **待审批列表 API (GET /api/tasks/pending)**：
  - 超级管理员：返回所有待审批记录（上帝视角）。
  - 普通管理员：仅返回 `tasks.reviewer_id` 等于当前登录者 ID 的记录，或 `reviewer_id` 为空的公共记录。
- **直接发分接口 (POST /api/thumbs)**：
  - 允许任何 `admin` 角色调用。**强制拦截提取** JWT Token 中的当前管理员 ID，并写入 `thumbs_records` 表的 `admin_id` 字段中。

## 3. 前端视图与交互改造 (Frontend Views)
- **后台菜单显示**：当 `is_super_admin === false` 时，隐藏“用户管理”、“商品管理”等核心设置菜单，仅显示“发星星”、“任务管理”和“我的待审批”。
- **发布任务表单 (`AdminTasks.vue`)**：
  - 增加「专属审批人」下拉框（拉取 admin 列表）。文案提示：“若不选择，则所有舰长均可审批”。
  - **沙盒限制**：如果是普通管理员登录，隐藏或禁用该下拉框。
- **流水审计展示 (`MyPoints.vue` / `Dashboard.vue`)**：
  - 在积分流水的“原因”列旁新增“**赋能官**”/“**发放人**”展示，通过关联查询显示具体发分人（例：`积极参与团队活动 | 发放人：张老师 ✨`）。
