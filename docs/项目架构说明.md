## 二、数据模型与关系

```
users (1) ──────────────── (N) thumbs_records
  │                              given_by → users.id
  │
  ├── (1) ──────────────── (N) exchange_records
  │                              product_id → products.id
  │
  └── (1) ──────────────── (N) wishlists

products (1) ─────────────── (N) exchange_records
```

### 关键字段说明

**thumbs_records.thumb_type**

| 值 | 含义 | points 变化 |
|----|------|------------|
| `single` | 单星辰币 ⭐ | +1 |
| `double` | 双星辰币 🚀 | +5 |
| `deduction` | 红牌警告 ⚠️ | -1 |

**products.is_blind_box**：`1` 时为盲盒商品，兑换时后端从 `BLIND_BOX_PRIZES` 常量池随机抽取奖品，结果写入 `exchange_records.remark`。

**wishlists.status**：`pending` → 审核中；`approved` → 批准并自动写入 `products`；`rejected` → 已拒绝。

## 三、核心业务流程

### 星辰币发放（含扣除）

```
管理员选择用户
    ↓
选择模式：发放 or 红牌警告（Switch 切换）
    ↓
发放：选择单星辰币(+1) / 双星辰币(+5)
扣除：固定 -1 能量
    ↓
填写原因（必填）+ 舰长寄语（选填）
    ↓
提交 → 后端验证管理员权限
    ↓
扣除时检查 available_points + points >= 0，不足则拒绝
    ↓
写入 thumbs_records（含 parent_message）
更新 users.total_points 和 users.available_points
    ↓
前端用户查看流水时，负数以红色显示，有寄语则展示提示条
```

### 商品兑换（含盲盒）

```
用户浏览商城，盲盒商品显示金色卡片 + 🎁 标签
    ↓
点击兑换 → 弹出确认对话框
    （盲盒：展示"神秘奖品"提示）
    ↓
提交 → 后端校验：已上架 · 库存充足 · 能量足够
    ↓
若 is_blind_box == True：
    random.choice(BLIND_BOX_PRIZES) → 写入 remark
    ↓
写入 exchange_records
扣除 users.available_points
减少 products.stock
    ↓
返回 message（盲盒：含抽奖结果）→ 前端 ElMessage 展示
```

### 星际心愿单流程

```
用户提交心愿（title + expected_points）
    ↓
写入 wishlists，status = 'pending'
    ↓
管理员在"心愿审核"页查看待审核列表
    ↓
批准 → 自动在 products 写入新商品（下架状态）
        wishlist.status = 'approved'
        管理员前往商品管理手动上架
拒绝 → wishlist.status = 'rejected'
    ↓
用户在心愿列表查看最新状态
```

### 兑换取消

```
管理员或用户发起取消
    ↓
校验：仅 completed 状态可取消，且本人或管理员
    ↓
users.available_points += record.points_spent（退回）
products.stock += record.quantity（退回）
record.status = 'cancelled'
```

## 四、技术选型

| 层级 | 选型 | 理由 |
|------|------|------|
| 前端框架 | Vue 3 + Vite | 组合式 API，构建速度快 |
| UI 库 | Element Plus | 组件丰富，与 Vue 3 原生契合 |
| 状态管理 | Pinia | Vue 3 官方推荐，轻量直观 |
| 图表 | ECharts | 功能完整，定制性强 |
| 后端框架 | Flask | 轻量灵活，适合中小型项目 |
| ORM | SQLAlchemy | Python 生态标准，防 SQL 注入 |
| 认证 | Flask-JWT-Extended | 无状态 Token，易于横向扩展 |
| 数据库 | MySQL 8.0 | InnoDB 事务支持，utf8mb4 字符集 |
| 容器化 | Docker + Nginx | 前端多阶段构建，生产部署标准化 |

## 六、安全设计

| 安全点 | 实现方式 |
|--------|---------|
| 密码存储 | Werkzeug `generate_password_hash`（scrypt） |
| 接口认证 | JWT Bearer Token，24 小时有效 |
| 权限校验 | 前端路由守卫 + 后端每个敏感接口单独校验 `role == 'admin'` |
| SQL 注入 | 全程使用 SQLAlchemy ORM，无裸 SQL 拼接 |
| XSS | Vue 模板自动转义，Element Plus 组件安全渲染 |
| CORS | Flask-CORS 全局启用，生产可收窄 origins |
| 文件上传 | 扩展名白名单 + 5MB 大小限制 + UUID 重命名防路径穿越 |
| 扣分保护 | 后端校验 `available_points + points >= 0`，防止能量为负 |

## 七、扩展建议

以下功能为合理的下一步方向，与现有架构无冲突：

- **积分排行榜**：基于 `users.total_points` 排序，新增一个 API 即可
- **操作日志**：新增 `operation_logs` 表，在关键操作写入记录
- **消息通知**：发放星辰币时向用户推送站内信，新增 `notifications` 表
- **数据导出**：兑换/流水记录导出 Excel，后端引入 `openpyxl`
- **商品分类**：新增 `categories` 表，`products` 表加外键
- **积分过期**：`thumbs_records` 加 `expires_at` 字段，定时任务扫描清零
- **移动端适配**：现有 CSS 已有 `@media` 响应式基础，进一步完善即可
