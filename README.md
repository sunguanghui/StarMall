# 🚀 星途补给站 (StarMall)

> 一款专为现代家庭打造的「星际探索」主题亲子成长与沉浸式奖励系统。
> 将家庭教育与游戏化机制深度融合，配合局域网 IoT 智能音箱的语音反馈，让孩子的每一步成长都充满仪式感。系统支持纯本地化 Docker 部署，拥有极其严密的多级权限与安全防御机制，做你家庭专属的“私人星际舰队”。

---

## 🌟 核心特性

### 🎮 游戏化成长引擎

* **星辰币奖惩体系**：支持灵活的单/双倍星辰币发放，以及梯度的视觉化惩罚（陨石撞击、黑洞吞噬），可用能量最低归零，不产生负债感。自带「15分钟后悔药」容错回滚机制。
* **碎片掉落与飞船升级**：打卡审批附带 30% 碎片掉落率，集齐碎片可升级专属飞船，极大提升日常任务的黏性。
* **动态连击与衰减机制**：连续 7 天完成任务触发「星云爆发」额外加分；连续 3 天未活跃触发能量衰减，培养持续的好习惯。
* **双向心愿流转**：除常规的能量商城商品（含概率盲盒）外，孩子可通过「星际心愿单」向舰长许愿，审批通过后自动转化为商城货架专属商品。

### 🔊 沉浸式星际广播台 (IoT 硬件联动)

系统内置 WebSocket 服务，支持与局域网内的智能音箱（如定制化 R1/软路由音箱）进行深度指令联动。

* **即时行为播报**：发分或审批通过时，音箱瞬间响应。播报文案自带完整社交语境（如：*“领航员爸爸，为小宇航员彤彤，发放了1枚星辰币！当前可用星辰币余额为：100 枚。”*），将情绪价值拉满。
* **定时星际简报**：内置高精度定时任务调度器，每日早晚准点自动盘点全站或特定宇航员的能量概况与心愿进度，并推送到音箱进行全屋广播。

### 🛡️ 银行级系统安全与鉴权

* **登舰审批流**：外网访客通过“登舰控制台”注册后，将被底层 API 强行剥夺越权可能（防批量赋值漏洞），强制处于 `pending` 待审批状态，必须由后台放行后方可入内。
* **儿童免密安全通道**：为小宇航员打造专属 3D 翻转卡片登录页。采用单向哈希加密（Hash String 255）存储图形序列，彻底剔除 `username` 等敏感字段的下发与暴露。
* **精细化时间基准**：全栈强制对齐北京时间（UTC+8），航行日志与审计流水精确到秒（`YYYY-MM-DD HH:mm:ss`），杜绝任务签到与定时播报的时区错乱。

## 👥 舰队角色与权限架构

系统采用严格的 RBAC 三级权限隔离机制，防止管理权限蔓延：

| 角色 | 身份标识 | 权限边界 |
| --- | --- | --- |
| **👑 舰长** | 超级管理员 | 掌握 100% 绝对控制权。可配置底层音箱网络参数、跨级提拔家庭成员角色、处理入伍审批。 |
| **👨‍🚀 领航员** | 普通管理员 | 拥有完整的日常业务权（发分、撤销、任务审批、发货）。**无法**触碰系统级设置，无法提拔其他管理员。 |
| **🧑‍🚀 星际探索者** | 小宇航员 | 仅可通过前端加密通道登录。拥有查看个人能量面板、提交任务、兑换商品与许愿的权限。 |

## 🛠️ 技术架构

本系统采用前后端分离架构，专为高效、安全的私有云/软路由部署而设计：

* **前端应用**：`Vue 3.4` + `Vite 5.0` + `Element Plus` (响应式 UI 与 ECharts 数据可视化)
* **后端服务**：`Flask 3.0` + `SQLAlchemy` + `JWT Auth` (支持高并发悲观锁防超扣)
* **定时与调度**：`APScheduler 3.10` (动态任务调度与每日凌晨数据库冷备)
* **数据与存储**：`MySQL 8.0` + 持久化数据卷卷挂载
* **容器化支持**：原生支持 `Docker Compose` 一键编排部署

## 🚀 部署指南 (Docker)

系统推荐使用 Docker 进行私有化部署，确保运行环境的高度一致性。

### 1. 环境准备

确保你的服务器或软路由（如 Debian/NanoPi 系统）已安装 `docker` 与 `docker-compose`。

### 2. 获取代码与配置

```bash
git clone <你的项目仓库地址> StarMall
cd StarMall
# 复制环境变量模板并修改数据库密码等配置
cp backend/.env.bak backend/.env

```

### 3. 一键登舰

```bash
docker-compose up -d

```

*首次启动时，容器会自动执行 `init.sql` 初始化数据库表结构。*

### 4. 访问系统

* **前端通用控制台**：打开浏览器访问 `http://<服务器IP>:28000`
* **初始默认设置**：请使用第一个注册的账号登入系统，该账号可在数据库中手动标记为 `is_super_admin=True` 以获取最高舰长权限。

## 🕹️ 核心操作指引 (新手舰长必读)

1. **新成员入伍**：邀请家人在主界面的“登舰控制台”完成注册。
2. **资质审批**：使用舰长账号登录后台，进入【用户管理】，对带有 `待审批` 标签的新成员点击 **[批准登舰]**。
3. **授予军衔**：在用户列表中点击“编辑”，将配偶或长辈的角色从“普通用户”修改为“管理员”，即刻完成领航员的任命。
4. **唤醒音箱**：进入左侧【星际广播台】（仅舰长可见），填入局域网智能音箱的 IP 与端口，开启全局播报开关，即可开启全沉浸式家庭互动。
