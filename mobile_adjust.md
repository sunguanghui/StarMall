# 角色与目标
你现在是 Claude Code。系统「星途补给站」的后端逻辑和童趣化 UI 已经完成。由于这套系统经常会在手机或平板端被一年级的小朋友及家长频繁使用，现在的核心任务是：**进行深度的移动端响应式适配 (Mobile Responsive Adaptation)**。

你需要通过编写 CSS 媒体查询 (`@media`)、调整 Element Plus 的响应式属性 (如 `xs`, `sm`) 或修改组件结构，让系统在小屏幕设备上依然保持完美、流畅的触控体验。

# 📱 移动端适配改造指南 (聚焦 frontend/src 目录)

## 1. 导航栏重构 (Mobile Navigation)
- 桌面端的侧边栏 (`el-menu` sidebar) 在手机屏幕（`max-width: 768px`）上极其占用横向空间。
- **改造要求**：在小屏幕下隐藏侧边栏，改用**底部导航栏 (Bottom Tab Bar)**，附带清晰的图标；或者将其改为顶部折叠式的**汉堡菜单 (Drawer)**，方便孩子单手点击。

## 2. 卡片与网格的流式布局 (Fluid Grid & Stacking)
- **Dashboard (仪表盘)**：桌面端并排显示的 4 个统计数据卡片，在手机上必须改为垂直单列堆叠 (`flex-direction: column`)，或者一行两列的紧凑布局，防止文字挤压变形。
- **商城列表 (Mall)**：商品卡片在移动端应自动响应，转为 `单列` 展示，确保商品图片在手机屏幕上依然大且清晰。

## 3. 痛点改造：表格响应式处理 (Responsive Tables)
- Element Plus 的 `el-table`（如：积分流水、兑换记录、后台用户管理等）在手机屏幕上会导致严重的横向溢出。
- **改造要求**：针对移动端，请给表格容器增加横向滚动限制 (`overflow-x: auto`)；对于用户端的流水记录，最好在移动端下隐藏原生 `el-table`，利用 `v-if/v-show` 降级渲染为一个**卡片列表 (Card List)** 以获得最佳体验。

## 4. 触控区域与间距优化 (Touch Targets & Spacing)
- **防误触**：移动端所有可点击的按钮 (`el-button`) 和输入框，高度必须适度增加（确保触摸区域至少满足 `44px` 高度），方便小朋友的小手操作。
- **空间利用**：通过媒体查询，大幅减小移动端下的外边距 (Margin) 和页面内边距 (Padding)，把宝贵的屏幕空间留给核心内容。
