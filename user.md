# 🛠️ 核心开发需求明细

## 需求 1：星际航行指南（新增帮助页）
- **内容来源**：请读取当前项目根目录下的 **`用户操作手册.md`** 文件，将其内容提取出来。
- **前端实现**：
  - 在 `frontend/src/views/` 目录下新建一个帮助页组件（建议命名为 `Help.vue`，页面标题为「**星际航行指南**」）。
  - 不要直接丢出纯文本。请使用 Element Plus 的组件（如 `el-collapse` 折叠面板、`el-card`、`el-timeline` 等）以及响应式排版，将操作手册的内容结构化、童趣化地渲染出来，方便孩子和家长阅读。
- **导航联动**：在侧边栏导航（`MainLayout.vue`）中，增加一个「**航行指南**」或「**帮助中心**」的菜单项，图标使用 Element Plus 的 `<QuestionFilled />` 或 `<InfoFilled />`，点击后路由跳转至该页面。

## 需求 2：个性化宇航员头像（用户头像系统）
- **数据库修改**：在 `users` 表中新增 `avatar` (VARCHAR) 字段，用来保存头像的文件名或预设的卡通头像标识。
- **核心逻辑与功能**：
  - **内置卡通太空头像（推荐/首选）**：在前端提供 4-6 个系统内置的、可爱的太空/卡通宇航员头像供用户一键点击更换。
  - **自定义本地上传（可选/增强）**：在用户修改头像的界面，支持点击上传本地图片作为头像。可以复用后端已有的图片上传逻辑，将头像图片安全地保存在 `backend/uploads/avatars/` 目录下。
- **UI/UX 展示优化**：
  - **个人中心入口**：用户（无论是管理员还是普通用户）点击右上角顶栏的个人姓名或默认头像时，弹出一个美观的弹窗（`el-dialog`）或跳转至设置页，展示当前的账号信息并提供修改头像的区域。
  - **全局实时刷新**：用户更换头像成功后，必须立即通过 **Pinia Store** 状态管理，让系统右上角顶栏、侧边栏（如果有展示）的头像同步更新，无需手动刷新网页。
  - **默认兜底**：如果新创建的用户没有设置头像，系统需默认展示一个可爱的卡通宇航员或星星图标作为默认头像。
